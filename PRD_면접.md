# PRD_TEMPLATE.md - 면접 질문 생성 툴

## 1. 기능 요구사항 (Functional Requirements)

- Gradio 입력창에 직무와 지원 자료(자소서/포트폴리오)를 입력하면 면접 질문이 생성된다.
- 질문 유형 5개 중 입력에 맞는 질문을 생성한다.
- 각 질문에는 질문 의도, 평가 포인트, 답변 가이드가 모두 포함된다.
- 부족한 입력에 대해 추가 질문을 반환한다.

---

## 1.1 입력값

사용자는 챗봇에게 다음 정보를 자연어로 입력할 수 있다.

- 지원 직무
- 지원 기업명
- 자소서
- 포트폴리오/프로젝트 경험
- 질문 유형 선호
- 난이도

예시:

```text
KB국민은행 디지털 직무 지원이에요. 자소서에 고객 데이터 분석 프로젝트 경험을 썼습니다. 직무 질문 위주로 받고 싶어요.
```

```text
네이버 백엔드 지원입니다. 대규모 트래픽 처리 프로젝트를 했고, 압박 질문도 연습하고 싶습니다.
```

---

## 1.2 출력값

툴은 사용자 입력을 바탕으로 다음 정보를 제공한다.

- 면접 질문 (유형별)
- 각 질문의 의도
- 평가 포인트
- 답변 가이드

기본 응답 형식:

```text
입력하신 직무와 자료를 기준으로 면접 질문을 생성했습니다.

1. 질문
- 질문 의도:
- 평가 포인트:
- 답변 가이드:

2. 질문
- 질문 의도:
- 평가 포인트:
- 답변 가이드:

3. 질문
- 질문 의도:
- 평가 포인트:
- 답변 가이드:

마무리 조언:
```

---

## 1.3 질문 유형 범위

MVP에서 생성하는 면접 질문 유형은 다음 5개로 제한한다.

| ID | 질문 유형 | 주요 설명 |
|---|---|---|
| `personality` | 인성 질문 | 가치관, 태도, 조직 적합성 파악 |
| `job_competency` | 직무 질문 | 지원 직무 역량·전문성 검증 |
| `project` | 프로젝트 경험 질문 | 자소서·포트폴리오 기반 경험 검증 |
| `company_fit` | 기업 맞춤 질문 | 기업 인재상·산업 특성 반영 |
| `pressure` | 압박 질문 | 돌발 상황 대처·논리 방어력 평가 |

## 2. 질문 생성 로직

### 2.1 1차 생성 방식

사용자의 입력 문장에 포함된 신호를 기준으로 질문 유형별 적합도를 계산한다.

| 사용자 입력 신호 | 우선 생성 질문 유형 |
|---|---|
| 자소서, 포트폴리오, 프로젝트 경험 | 프로젝트 경험 질문 |
| 직무 명확(데이터, 개발, 기획, 분석) | 직무 질문 |
| 지원 기업명 입력됨 | 기업 맞춤 질문 |
| 가치관, 성향, 협업, 태도 | 인성 질문 |
| 고난이도, 실전, 압박 요청 | 압박 질문 |

### 2.2 점수 계산 규칙

1. 사용자 입력(직무, 자소서, 포트폴리오, 기업명)을 정규화한다.
2. 입력 신호별 질문 유형 적합도 점수를 더한다.
3. 자소서·포트폴리오 내용이 지원 직무와 맞으면 가중치를 더한다.
4. 유형별 우선순위에 따라 질문을 생성한다.
5. 직전 답변 점수로 다음 질문 난이도를 조정한다.
   - 80점 이상이면 난이도 상승
   - 60~79점이면 난이도 유지
   - 59점 이하면 난이도 하향

### 2.3 부족한 입력 판단

다음 조건 중 2개 이상에 해당하면 생성 대신 추가 질문을 반환한다.

- 입력 길이가 20자 미만
- 직무, 자소서, 포트폴리오 중 어떤 정보도 없음
- 면접·직무와 관련된 키워드가 없음
- 사용자의 질문이 면접 준비와 무관함
