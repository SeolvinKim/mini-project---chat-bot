# PRD_TEMPLATE.md - 자소서 피드백 툴

## 1. 기능 요구사항 (Functional Requirements)

- Gradio 입력창에 자소서 본문과 문항을 입력하면 피드백이 출력된다.
- 자소서를 6개 평가 항목 기준으로 진단한다.
- 항목별 진단, 강점/약점, 문장 수정 제안이 모두 포함된다.
- 부족한 입력에 대해 추가 질문을 반환한다.

---

## 1.1 입력값

사용자는 챗봇에게 다음 정보를 자연어로 입력할 수 있다.

- 자소서 문항
- 자소서 본문
- 지원 직무
- 강조하고 싶은 경험
- 보유 역량
- 글자 수 제한

예시:

```text
지원동기 문항입니다. "저는 어릴 때부터 금융에 관심이 많았고, 열정을 가지고 다양한 활동에 참여했습니다." 은행 디지털 직무 지원이에요.
```

```text
이 자소서 봐주세요. 증권사 백엔드 지원입니다. "대학 프로젝트에서 팀장을 맡아 책임감을 길렀습니다."
```

---

## 1.2 출력값

툴은 사용자 입력을 바탕으로 다음 정보를 제공한다.

- 자소서 총평
- 항목별 진단(강점/약점)
- 문장 단위 수정 제안
- 앞으로의 보완 방향

기본 응답 형식:

```text
현재 자소서를 기준으로 항목별 피드백을 드립니다.

1. 평가 항목
- 진단:
- 강점:
- 개선 방향:

2. 평가 항목
- 진단:
- 강점:
- 개선 방향:

3. 평가 항목
- 진단:
- 강점:
- 개선 방향:

마무리 조언:
```

---

## 1.3 평가 항목 범위

MVP에서 진단하는 자소서 평가 항목은 다음 6개로 제한한다.

| ID | 평가 항목 | 주요 설명 |
|---|---|---|
| `intent` | 문항 부합 | 묻는 질문에 실제로 답하고 있는지 검토 |
| `structure` | 두괄식/STAR | 결론 우선, 상황-과제-행동-결과 구조인지 검토 |
| `relevance` | 직무 연관성 | 지원 직무 역량과 본문이 연결되는지 검토 |
| `specificity` | 구체성/정량화 | 추상어 대신 숫자·사례가 있는지 검토 |
| `uniqueness` | 차별성 | 누구나 쓸 문장인지, 본인만의 경험인지 검토 |
| `readability` | 가독성 | 문장 길이, 클리셰, 오탈자 검토 |

---

## 2. 피드백 로직

### 2.1 1차 진단 방식

사용자의 자소서 문장에 나타난 신호를 기준으로 항목별 점검 우선순위를 계산한다.

| 자소서에 나타난 신호 | 우선 점검 항목 |
|---|---|
| 숫자, %, 정량 표현 없음 | 구체성/정량화 |
| 첫 문장이 결론이 아님, 결과 누락 | 두괄식/STAR |
| 지원 직무 역량·키워드 미등장 | 직무 연관성 |
| 열정, 노력, 책임감 등 추상어 빈발 | 구체성/정량화, 차별성 |
| 문항과 본문 주제 불일치 | 문항 부합 |
| 한 문장 80자 이상, 클리셰 | 가독성 |

### 2.2 점수 계산 규칙

1. 자소서 입력을 문항과 본문으로 정규화한다.
2. 항목별 약점 신호가 발견되면 우선순위 점수를 더한다.
3. 지원 직무, 경험이 본문과 맞으면 직무 연관성에 가중치를 더한다.
4. 약점이 큰 항목부터 상세 피드백한다.
5. 동점이면 다음 우선순위를 적용한다.
   - 합격에 더 치명적인 항목(직무 연관성, 구체성)
   - 사용자가 바로 고칠 수 있는 항목(가독성, 정량화)
   - 본인 경험으로 보강 가능한 항목

### 2.3 부족한 입력 판단

다음 조건 중 2개 이상에 해당하면 피드백 대신 추가 질문을 반환한다.

- 자소서 본문이 100자 미만
- 지원 직무, 문항 정보가 전혀 없음
- 자소서가 아닌 입력(질문/잡담)
- 무엇을 봐달라는지 불명확함
